Phase 0 · ~2–4 weeks · personal-tool harden

Phase 0 — stop the single-user assumptions from leaking

Tenant isolation & auth

  • Ship account deletion flow (ClickUp 86e0vjvmw). Cascade through every userId-scoped table.
  • Add email change + verification flow.
  • Audit every db.select / update / delete for userId filter.
  • Add cross-user tenant-isolation tests; gate CI on them.
  • Add rate limiting on auth and key endpoints.
  • Decide API key scoping for v1 (full account vs read-only vs per-resource).
  • Rotate the production API key out of CLAUDE.md; move secrets to 1Password or equivalent.

Observability (self-hosted)

  • Stand up an observability VPS (separate from app host).
  • Deploy GlitchTip; wire Sentry SDK on client + server.
  • Deploy Loki + structured JSON log shipping from the Node process.
  • Deploy Prometheus + node-exporter + prom-client for app metrics.
  • Deploy Grafana with dashboards for errors, logs, basic uptime.
  • Deploy Uptime Kuma; set alert channels (email/Slack).
  • Add log-redaction check to the structured logger.

Transactional email (self-hosted)

  • Deploy Postal (or stand up Postfix + DKIM) for transactional SMTP/API.
  • Wire password-reset email through it.
  • Wire verification + signup-confirmation through it.

Ops basics

  • Verify Postgres backup schedule + try a restore once.
  • Verify HTTPS / HSTS / CSP headers are set on the host.
  • Enable Dependabot or Renovate for dependency updates.

Phase 1 · ~4–6 weeks · closed beta

Phase 1 — survivable for 10–25 friendly strangers

Marketing site (= unauthed PWA routes)

  • Split route groups: (marketing) vs (app).
  • Ensure SSR works for unauthed routes (no authed-only client state imported).
  • Build skeleton landing page with email capture (problem block, approach block, CTA).
  • Set up Listmonk for newsletter / waitlist.
  • Send the first monthly progress update, even to 5 people.

Onboarding & empty states

  • Build sign-up + first-run wizard (give me one stock entry + one recipe so the dashboard isn't empty).
  • Add "Add starter pantry" seed-data button.
  • Add loading states across the app (ClickUp 86e0zjy0r).
  • Write user-facing help docs (5 short pages: stock, products, recipes, meal plans, quickstart).

Support & bugs

  • Set up support@<domain> routing to your inbox.
  • Add in-app "report an issue" link (grab URL + screenshot).
  • Write customer support playbook (where tickets go, response-time commitment, canned responses).

Legal floor

  • Generate Terms of Service (Termly or iubenda).
  • Generate Privacy Policy (PIPEDA-aware).
  • Generate Cookie consent + banner if any non-essential cookies.
  • Add a beta-period disclaimer ("service is in beta and may break").

Feature flagging + closed beta

  • Add a boolean betaAccess column on user (or similar) for staged rollout.
  • Recruit 10–25 friendly strangers (not friends) for closed beta.
  • Run weekly check-in with beta cohort (async via Listmonk or a private channel).

Phase 2 · ~3–6 weeks · open beta, free

Phase 2 — public signup, still free, no payments yet

Friction reduction

  • Add Google OAuth sign-in.
  • Add pagination + lazy loading for product / recipe / stock-log lists.
  • Mobile parity pass: every flow works on a phone.
  • Add PWA install prompt.

Critical-path features for ICP A (value-conscious)

  • Ship receipt scanning v1 (hosted OCR behind feature flag).
  • Surface waste-reduction signal on the dashboard ("you have $X of food expiring this week").

Analytics & cost controls

  • Deploy self-hosted Plausible (or Umami) for product analytics.
  • Instrument the signup → first-stock-entry → cook → 7-day-return funnel.
  • Set hosting cost monitoring + an alert threshold for unexpected spikes.

Marketing surface + paid prep

  • Add newsletter signup on the marketing site.
  • Draft 3–5 paid ad creative concepts (static + short video).
  • Test ad creative against waitlist before paid spend goes live.

Phase 3 · ~4–8 weeks · paid launch

Phase 3 — pricing, payments, take real money

Pricing & entitlements

  • Decide on the pricing number (leading thesis: annual subscription, CAD $24–48/yr).
  • Integrate Stripe Checkout (CAD-primary, Stripe Tax on).
  • Add plan column on user for entitlements.
  • Wire Stripe webhooks → entitlement table (idempotent).
  • Decide beta-cohort grandfathering (recommended: free year for first 50–100).
  • Build pricing page on the marketing site.
  • Write refund policy (recommend 14-day no-questions) and place it on the marketing site.

Ops readiness

  • Set calendar reminder for HST/GST registration at projected $25k revenue.
  • Set up monthly bookkeeping cadence (Wave reconciliation, expense check, HST threshold check).
  • Write incident response runbook (auth down / payments down / DB down / host down / you're sick).
  • Write risk register one-pager (top 8–10 risks + planned response).

Launch mechanics

  • Soft-launch to waitlist 72 hours before public; founder-cohort offer attached.
  • Turn on paid Meta ads in test mode ($20–$50/day, 3–5 creatives).
  • Turn on paid TikTok Spark ads (same scope, paired with creator content).
  • Launch one community post on r/MealPrepSunday or r/EatCheapAndHealthy (value-led, not promo-shaped).
  • Post launch announcement on BlueSky + Indie Hackers.
  • Reply to every comment / DM in the first 48 hours.
  • Keep the support inbox open all day for the first week.

Phase 4 · ongoing · steady state

Phase 4 — run a small business

Cadence

  • Weekly time block for support email triage (hard cap so it doesn't eat the build).
  • Monthly sustainability check-in (am I enjoying this / sleeping / partner okay / learning).
  • Quarterly metric review against the matters-vs-doesn't-matter list.
  • Annual security review checklist.

Growth wave 2

  • Reach out to 5 of the 10 identified content creators with a flat-fee + free-year offer.
  • Pitch BlogTO / Toronto Star lifestyle desk on the "Toronto-built tool to reduce food waste" angle.
  • Start long-tail SEO content cadence (1 post/month minimum).

Product evolution

  • Public changelog / "what's new" page.
  • Reassess pricing after first 100 paid users.
  • Decide whether to ship the self-host SKU for ICP B.
  • Decide on Quebec Law 25 compliance work if Quebec sign-ups appear.

Business evolution

  • Reassess incorporation triggers quarterly (revenue ≥ $50k/yr, first hire, contract conflict, liability exposure).
  • Get E&O / cyber liability insurance quote once incorporation lands.
  • Cachet status page deployment (only when outages start affecting paying customers).

Eventually but not urgent

  • Write product north star ("if this works wildly in 5 years, what does it look like?").
  • Start a decision log (single markdown file, append-only, why-did-I-do-X).
  • Press kit (boilerplate + logos + screenshots + founder quote).
  • Content calendar / email cadence schedule.

Explicitly not doing

  • Hacker News launch (wrong audience for the lead ICP).
  • VC pitch deck.
  • Native mobile app (PWA first; revisit at 100+ paying users).
  • Detailed multi-year financial projections.
  • Physical office (until a partnership specifically demands one).
