Spec: Quick-extend stock expiry date

Kitchen Manager · Draft

Drafted2026-05-22
Shipped2026-05-22

Problem

Expiry dates on stock entries are often estimates. You set "good for 5 days" when you bought the leftovers, then on day 5 the dish is clearly still fine. Today the only way to push the date is to click the date, type an absolute new date into an OS date picker, and dismiss. That's three interactions for a 1-day nudge.

Goal

Make extending an expiry date a single click for the common deltas (+1d, +3d, +1w, +2w, +1mo), without removing the ability to pick a specific date.

UX options

Option 1: Dedicated "extend" button + chip popover  (recommended)

Add a second small affordance next to the existing pencil edit. A ⏱+ icon opens a popover containing chip presets. Clicking a chip applies the delta and closes the popover. Pencil-edit is unchanged.

Stock row, default state
Leftover lasagna Exp: May 27, 2026
Click the ⏱+ button
Extend expiry by…
+1 day +3 days +1 week +2 weeks +1 month

Pros: Discoverable; the verb "extend" matches the user's mental model; doesn't disturb the existing edit flow; chip layout reads as a menu of common nudges.
Cons: Two affordances per row instead of one.

Option 2: One popover via the pencil — chips + calendar

Collapse both flows into one. The existing pencil/date click opens a popover with chip presets on top and the date input below. Saves one icon per row but costs one extra click for absolute edits (popover → calendar → pick).

Stock row, default state
Leftover lasagna Exp: May 27, 2026
Click the date
Extend expiry by…
+1 day +3 days +1 week +2 weeks +1 month
— or pick a date —

Pros: One affordance, less visual clutter; both flows live in one place.
Cons: Slower for the user who actually wants to pick a date; changes existing behaviour (click date no longer focuses input directly).

Option 3: Inline +/− day stepper

Two tiny buttons flanking the date that step by one day. Cheapest for ±1 day but tedious for "+1 week" and noisy with two extra controls per row.

Pros: Zero-popover; immediate feedback.
Cons: Bad fit for week+ extensions; doubles the controls per row; risk of accidental clicks on dense rows.

Recommendation: Ship Option 1. It matches the user's language ("extend"), preserves the existing edit-absolute flow, and the chip popover scales cleanly if we want to add new presets later. Anchor the delta from max(currentExpiry, today) so already-expired entries push to a useful future date rather than further into the past.

Behaviour details

  • Anchor date: deltas are applied to max(currentExpiry, today). So extending a January-expired entry on May 22 with +1 week produces May 29, not Jan-something.
  • No current expiry: presets still work; anchor is today.
  • Toast: reuse the existing toast.success shape from the editor: Expiry set to May 28, 2026.
  • Close behaviour: popover dismisses on outside click, Escape, or after a chip click.
  • Keyboard: Tab into the popover should iterate chips; Enter triggers the focused chip.

Decisions you need to make

  1. Preset set: do +1d / +3d / +1w / +2w / +1mo cover your real-world cases, or do you want different defaults (e.g. +2d instead of +3d, or add +3mo)?
  2. Anchor rule: max(currentExpiry, today) as above, or always anchor from current expiry (which can keep an expired item in the past after a "+1 day" tap)?
  3. Shorten too? Mirror chips for −1d / −3d / −1w in case you over-estimated? Or keep this strictly an "extend" feature?
  4. Reach: apply to every place ExpiryDateEditor is used (stock list, stock detail, dashboard expiring panel), or only the stock entry row? Default: all, since it's the same component.

Tests

  • Unit: a helper addDelta(expiry, today, deltaDays) that returns the new date string, with cases for "no expiry", "expiry in the future", "expiry in the past".
  • Component: clicking +1 week calls updateStockEntry.mutateAsync with the expected ISO date.
  • Component: popover closes after a chip click and on Escape.

Out of scope

  • "Smart" extend suggestions based on product type or past behaviour. Static presets only.
  • Bulk-extend across multiple stock entries at once.
  • Editing a stock entry's purchase date — separate field, same pattern could apply later.
